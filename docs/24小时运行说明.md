# 24 小时运行 / 开机自启说明

让另一台设备随时通过浏览器访问 **http://你的本机IP:5173** 进行测试。

---

## 对外暴露的接口一览

| 服务       | 端口 | 本机访问           | 局域网访问              | 外网访问说明           |
|------------|------|--------------------|-------------------------|------------------------|
| 前端页面   | 5173 | http://localhost:5173 | http://本机IP:5173   | 需端口转发或隧道       |
| 后端 API   | 8000 | http://localhost:8000 | http://本机IP:8000   | 同上                   |
| Gradio UI  | 7860 | http://localhost:7860 | http://本机IP:7860   | 同上（若需可单独暴露） |

本机与前端已配置为监听所有网卡（0.0.0.0 / host: true），同一 WiFi 下用「局域网访问」即可；**外网**需按下面「二、另一台设备怎么访问 → 外网访问」配置端口转发或隧道。

---

## 一、本机需要做的设置

### 1. 防止 Mac 睡眠（必做）

- **系统设置** → **锁定屏幕**（或 **电池**）  
  - 接电源时：将「关闭显示器」和「防止自动休眠」设为 **永不**（或足够长时间）  
- 这样电脑不会睡死，服务才能一直在线。

### 2. 安装「登录时自动启动」服务（推荐）

在终端执行（**一行一行复制执行**）：

```bash
# 复制 plist 到当前用户的开机自启目录
cp "/Users/criss/Desktop/机器学习学习资料/石若含毕业项目/vestibular_pose/scripts/com.vestibular.website.plist" ~/Library/LaunchAgents/

# 加载服务（本次登录立即生效，下次开机也会自动启动）
launchctl load ~/Library/LaunchAgents/com.vestibular.website.plist
```

之后每次**登录 Mac**，后端（8000）和前端（5173）都会自动在后台运行。

**查看是否在运行：**

```bash
launchctl list | grep vestibular
# 若显示 com.vestibular.website 且前面数字为 0，表示正在运行
```

**停止自启、卸载服务：**

```bash
launchctl unload ~/Library/LaunchAgents/com.vestibular.website.plist
```

### 3. 不装自启时：手动开 24 小时

不复制 plist 也可以，每次需要 24 小时服务时，在终端执行：


不要关这个终端窗口，服务会一直跑。要停掉时在该窗口按 `Ctrl+C`，或另开终端执行：

```bash
bash "/Users/criss/Desktop/机器学习学习资料/石若含毕业项目/vestibular_pose/scripts/stop_background.sh"
```

## 二、另一台设备怎么访问

### 同一 WiFi（局域网）

- 浏览器打开 **http://192.168.1.13:5173**（若本机 IP 变了，先在本机运行 `ipconfig getifaddr en0` 看当前 IP）
- 本机：**http://localhost:5173**

### 外网访问（不在同一 WiFi 的设备，通过公网访问）

你的服务已监听 `0.0.0.0`，本机与路由器不拦的话，外网要能访问还需要下面**二选一**：

---

#### 方式一：路由器端口转发

1. 登录路由器管理页（如 192.168.1.1）。
2. 找到「端口转发 / 虚拟服务器 / NAT」。
3. 新增两条规则：

   | 外部端口 | 内部 IP（你 Mac 的局域网 IP） | 内部端口 | 协议 |
   |----------|------------------------------|----------|------|
   | 5173     | 192.168.1.13                 | 5173     | TCP  |
   | 8000     | 192.168.1.13                 | 8000     | TCP  |

4. 保存后，**任意网络**下的设备在浏览器访问：
   - 前端页面：**http://你的公网IP:5173**
   - 后端 API：**http://你的公网IP:8000**  
   （公网 IP 可在本机运行 `curl -s ifconfig.me` 查看；若运营商使用 CGNAT，可能无法用此方式。）

---

#### 方式二：内网穿透 / 隧道（推荐，不用动路由器）

用隧道工具在本地和公网之间建一条通道，外网通过**固定链接**访问你的 5173/8000。

**A. 使用 Cloudflare Tunnel（免费、无需公网 IP）**

1. 安装 cloudflared（任选一种）：
   - Homebrew：`brew install cloudflared`
   - 或到 [https://developers.cloudflare.com/cloudflare-one/connections/connect-apps/install-and-setup/installation/](https://developers.cloudflare.com/cloudflare-one/connections/connect-apps/install-and-setup/installation/) 下载
2. 在终端执行（先确保前端、后端已在运行）：
   ```bash
   # 暴露前端 5173，会得到一个公网 URL
   cloudflared tunnel --url http://localhost:5173
   ```
   终端里会打印一行 **https://xxx.trycloudflare.com**，把该链接发给对方即可在任意设备打开。

   若也要暴露 API（8000），再开一个终端：
   ```bash
   cloudflared tunnel --url http://localhost:8000
   ```
   会得到另一个 URL，供接口调用使用。

**B. 使用 ngrok**

1. 注册 [ngrok](https://ngrok.com)，安装并配置 auth token。
2. 执行：`ngrok http 5173`，用给出的 **https://xxx.ngrok.io** 访问前端；API 同理可开 `ngrok http 8000`。

**一键生成公网链接（本仓库脚本）**

在项目目录执行（会优先用 cloudflared，没有则用 ngrok）：

```bash
bash scripts/expose_tunnel.sh 5173
```

终端会打印一个公网 URL，把该链接发给对方即可在任意设备打开前端。关掉该终端或 Ctrl+C 后隧道断开。

**若出现「lookup _v2-origintunneld._tcp.argotunnel.com ... no such host」**：当前网络 DNS（如 198.18.x.x，常见于 VPN/代理）无法解析 Cloudflare。任选其一：(1) 将 Mac 的 DNS 改为 1.1.1.1（系统设置 → 网络 → Wi‑Fi → 详细信息 → DNS，添加 1.1.1.1）；(2) 使用 ngrok：`brew install ngrok` 后执行 `ngrok http 5173`。

---

**小结**：同一 WiFi 用 **http://本机IP:5173**；外网用**端口转发**（http://公网IP:5173）或**隧道**（把 cloudflared/ngrok 给的链接发给对方）。

## 四、若换了电脑或项目路径

需要改 plist 里的路径后再重新复制并 load：

- 用文本编辑器打开 `scripts/com.vestibular.website.plist`
- 把其中的 `/Users/criss/Desktop/机器学习学习资料/石若含毕业项目/vestibular_pose` 全部换成你当前的项目路径
- 保存后重新执行上面的 `cp` 和 `launchctl load` 命令
