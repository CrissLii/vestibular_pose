### 步骤3：组织目录结构

清洗后的视频需要按照训练脚本期望的结构组织。有两个选择：

#### 选项A：保持旧结构（最简单）

直接用清洗后的视频替换旧视频：

```bash
# 备份原始数据
mkdir -p dataset_backup
cp -r dataset/1.原地旋转5-10圈 dataset_backup/
cp -r dataset/2.原地向上跳跃 dataset_backup/
cp -r dataset/3.小推车 dataset_backup/
cp -r dataset/4.直线加速跑 dataset_backup/
cp -r dataset/5.前滚翻 dataset_backup/
cp -r dataset/6.抬头向上 dataset_backup/

# 用清洗后的视频替换
rm -rf dataset/1.原地旋转5-10圈/*
rm -rf dataset/2.原地向上跳跃/*
rm -rf dataset/3.小推车/*
rm -rf dataset/4.直线加速跑/*
rm -rf dataset/5.前滚翻/*
rm -rf dataset/6.抬头向上/*

cp dataset_cleaned/1.原地旋转5-10圈/* dataset/1.原地旋转5-10圈/
cp dataset_cleaned/2.原地向上跳跃/* dataset/2.原地向上跳跃/
cp dataset_cleaned/3.小推车/* dataset/3.小推车/
cp dataset_cleaned/4.直线加速跑/* dataset/4.直线加速跑/
cp dataset_cleaned/5.前滚翻/* dataset/5.前滚翻/
cp dataset_cleaned/6.抬头向上/* dataset/6.抬头向上/
```

#### 选项B：组织成新结构（更规范，但需要手动分类）

如果你的视频文件名能区分正面/侧面和受试者，可以组织成：

```
dataset/
├── 1.原地旋转/
│   ├── front/
│   │   ├── subject_01/
│   │   │   └── IMG_1262_trimmed.mp4
│   │   ├── subject_02/
│   │   │   └── IMG_1263_trimmed.mp4
│   │   └── subject_03/
│   │       └── IMG_1264_trimmed.mp4
│   └── side/
│       ├── subject_01/
│       │   └── IMG_2578_trimmed.mp4
│       └── subject_02/
```

**手动分类提示**：
- 查看视频文件名或拍摄时间
- 正面视角通常 IMG_12XX，侧面通常 IMG_25XX
- 按拍摄顺序分配 subject_01, subject_02, ...

## 💻 关于训练设备选择

### MacBook M1 vs 阿里云 2核4G

| 特性 | MacBook M1 | 阿里云 2核4G |
|------|-----------|-------------|
| CPU性能 | ⭐⭐⭐⭐⭐ 强 | ⭐⭐ 弱 |
| 内存 | 8GB+ | 4GB（可能不够） |
| GPU加速 | ✅ MPS加速 | ❌ 无GPU |
| 姿态估计速度 | 快（~20秒/视频） | 慢（~60秒/视频） |
| 训练速度 | 快 | 慢 |
| 网络 | 本地数据 | 需上传数据 |
| **推荐度** | ⭐⭐⭐⭐⭐ | ⭐⭐ |

**结论：强烈推荐用 MacBook M1**

原因：
1. **M1性能远超2核CPU**：M1的神经引擎和GPU加速对YOLO非常友好
2. **内存充足**：4GB服务器可能在处理视频时内存不足
3. **数据本地**：不需要上传167个视频（几十GB）
4. **调试方便**：本地可以随时查看结果

### 如何在M1上加速训练

确保使用MPS（Metal Performance Shaders）加速：

```bash
# 检查PyTorch是否支持MPS
python -c "import torch; print(f'MPS available: {torch.backends.mps.is_available()}')"

# 如果不支持，重新安装PyTorch
pip install --upgrade torch torchvision
```

## ⚠️ 注意事项

1. **备份数据**：清洗前务必备份原始视频
2. **检查磁盘空间**：清洗后的视频会占用额外空间（约10-20GB）
3. **分批处理**：可以先处理1-2个类别，验证效果后再继续
4. **保存缓存**：`data/kpt_cache.pkl` 很重要，不要轻易删除
