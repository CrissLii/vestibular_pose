# 阿里云 / 腾讯云服务器部署说明

在 Linux 服务器上部署 vestibular_pose 项目：**无需 Nginx**，由 FastAPI 直接提供前端静态资源与 API，使用 systemd 保持进程常驻。访问方式：**http://公网IP:8000**。

---

## 一、前置要求

- 已有一台云服务器（推荐 2 核 8G，Ubuntu 22.04 或 Alibaba Cloud Linux）
- 已把项目代码推送到 Git（GitHub / Gitee / 阿里云 Code 等）
- 能 SSH 登录到服务器

---

## 二、服务器环境准备

### 1. 用 root 或 sudo 用户 SSH 登录

```bash
ssh root@你的服务器公网IP
# 示例：ssh root@8.152.197.12
```

### 2. 安装依赖

先确认系统类型：执行 `cat /etc/os-release`，若出现 `Alibaba Cloud Linux`、`CentOS`、`Rocky` 等则用 **yum**；若为 `Ubuntu`、`Debian` 则用 **apt**。

**阿里云 / CentOS / Alibaba Cloud Linux（用 yum，apt 报错时用下面这组）：**

```bash
sudo yum install -y git python3 python3-pip
# 安装 Node 18+（前端构建需要）
curl -fsSL https://rpm.nodesource.com/setup_20.x | sudo bash -
sudo yum install -y nodejs
```

**Ubuntu / Debian（用 apt）：**

不需要安装 Nginx，只需：

```bash
sudo apt update
sudo apt install -y git python3 python3-venv python3-pip nodejs npm
```

若 `nodejs` 版本过旧（需 Node 18+），可改用 NodeSource：

```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
```

---

## 三、拉取代码并安装项目

### 1. 克隆仓库（请替换为你的仓库地址）

```bash
cd /opt
sudo git clone https://github.com/CrissLii/vestibular_pose.git
# 或 Gitee: https://gitee.com/你的用户名/vestibular_pose.git
sudo chown -R $USER:$USER /opt/vestibular_pose
cd /opt/vestibular_pose
```

### 2. 一键安装后端 + 构建前端

```bash
cd /opt/vestibular_pose
bash scripts/setup_server.sh
```

该脚本会：创建 Python 虚拟环境、安装后端依赖、创建数据目录、安装前端依赖并执行 `npm run build`。构建完成后，后端会自动提供 `frontend/dist` 中的页面，**无需 Nginx**。

### 3. 可选：手动分步执行

```bash
cd /opt/vestibular_pose
python3 -m venv .venv
source .venv/bin/activate
pip install -e .
mkdir -p data/raw data/outputs/videos data/outputs/reports
cd frontend && npm ci && npm run build && cd ..
```

首次运行时会自动下载 YOLO 权重（或把本地的 `yolo11n-pose.pt` 放到项目根目录）。

## 四、后端常驻：systemd 服务

后端需监听 **0.0.0.0:8000**，以便外网通过 **http://公网IP:8000** 访问（同一进程同时提供前端页面和 API）。

### 1. 新建服务文件

```bash
sudo cp /opt/vestibular_pose/deploy/vestibular-api.service.example /etc/systemd/system/vestibular-api.service
```

编辑（若项目不在 `/opt/vestibular_pose` 或不用 root 运行，需改路径和 User）：

```bash
sudo nano /etc/systemd/system/vestibular-api.service
```

确认 `ExecStart` 为 **--host 0.0.0.0 --port 8000**，例如：

```ini
[Unit]
Description=Vestibular Pose FastAPI Backend (前端+API 一体)
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/vestibular_pose
Environment="PATH=/opt/vestibular_pose/.venv/bin:/usr/local/bin:/usr/bin"
ExecStart=/opt/vestibular_pose/.venv/bin/uvicorn vestibular.api.server:app --host 0.0.0.0 --port 8000
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

### 2. 启动并开机自启

```bash
sudo systemctl daemon-reload
sudo systemctl enable vestibular-api
sudo systemctl start vestibular-api
sudo systemctl status vestibular-api
```

## 五、防火墙与安全

- 放行 **8000** 端口（本方案不用 80 端口，无需 Nginx）：

  **Ubuntu (ufw)：**

  ```bash
  sudo ufw allow 8000
  sudo ufw allow 22
  sudo ufw enable
  ```

  **CentOS / Alibaba Cloud（firewalld）：**

  ```bash
  sudo firewall-cmd --permanent --add-port=8000/tcp
  sudo firewall-cmd --permanent --add-service=ssh
  sudo firewall-cmd --reload
  ```

- **阿里云控制台**：安全组入方向放行 **8000**（和 22）。

- 浏览器访问：**http://你的公网IP:8000**（示例：http://8.152.197.12:8000）
- 更新代码后：

## 八、可选：使用 Nginx 反代到 80 端口

若希望用 **http://公网IP**（80 端口）访问而不用写 `:8000`，可选用 Nginx 做反向代理：

1. 安装 Nginx：`sudo apt install -y nginx`（或 yum）
2. 复制并编辑配置：  
   `sudo cp /opt/vestibular_pose/deploy/nginx-vestibular.conf.example /etc/nginx/sites-available/vestibular`  
   将 `proxy_pass http://127.0.0.1:8000` 保留，并设置 `server_name` 或 `listen 80 default_server`。
3. 启用站点并重载 Nginx，安全组放行 80。

此时访问 **http://公网IP** 即可，后端仍只需运行 uvicorn（systemd 不变）。
