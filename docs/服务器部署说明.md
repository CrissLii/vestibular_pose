## 二、接下来要做的 3 步

### 步骤 1：创建 systemd 服务文件（让后端开机自启、7×24 运行）

在服务器上**逐行执行**：

```bash
cat > /etc/systemd/system/vestibular-api.service << 'EOF'
[Unit]
Description=Vestibular Pose FastAPI Backend
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/home/vestibular_pose
Environment="PATH=/home/vestibular_pose/.venv/bin:/usr/local/bin:/usr/bin"
ExecStart=/home/vestibular_pose/.venv/bin/uvicorn vestibular.api.server:app --host 0.0.0.0 --port 8000
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF
```

### 步骤 2：启动服务并设置开机自启

```bash
sudo systemctl daemon-reload
sudo systemctl enable vestibular-api
sudo systemctl start vestibular-api
```

检查是否启动成功：

```bash
sudo systemctl status vestibular-api
```

看到 **active (running)** 表示成功。若看到 failed，用以下命令查看报错：

```bash
sudo journalctl -u vestibular-api -n 50 --no-pager
```

### 步骤 3：放行 8000 端口

**服务器防火墙（在服务器终端执行）：**

```bash
sudo firewall-cmd --permanent --add-port=8000/tcp
sudo firewall-cmd --reload
```

若提示 firewalld 未运行，则服务器端无需操作。

**阿里云控制台安全组（必须做）：**

1. 登录阿里云控制台 → **云服务器 ECS** → 找到你的实例
2. 点击实例 → **安全组** → 点击安全组名
3. **入方向** → **手动添加**
4. 填写：
   - 授权策略：**允许**
   - 优先级：**1**
   - 协议类型：**自定义 TCP**
   - 端口范围：**8000/8000**
   - 授权对象：**0.0.0.0/0**
5. **保存**

## 六、可选：用 80 端口访问（不写 :8000）

若希望用 **http://8.152.197.12** 直接访问（不带端口号），需额外安装 Nginx：

```bash
sudo yum install -y nginx
```

创建配置：

```bash
cat > /etc/nginx/conf.d/vestibular.conf << 'EOF'
server {
    listen 80;
    server_name 8.152.197.12;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_read_timeout 300s;
    }
}
EOF

sudo nginx -t && sudo systemctl enable --now nginx
```

然后阿里云安全组放行 **80** 端口，即可用 **http://8.152.197.12** 访问。
